## 1. Project Motivation（專案動機）

- 台灣口腔癌發生率長年位居男性癌症前五名，與「可預防」的生活習慣高度相關：根據國健署引用的本土研究（高雄醫學大學葛應欽教授團隊），吸菸者罹癌風險約為一般人 18 倍、嚼檳榔者約 28 倍、飲酒者約 10 倍；兩者並存（菸+檳榔）風險驟升至 89 倍，菸、檳榔、酒三者皆有（俗稱「香檳酒」）甚至高達 123 倍。
- 國健署提供 30 歲以上（原住民 18 歲以上）有吸菸或嚼檳榔習慣者每兩年一次免費口腔黏膜篩檢，花蓮慈濟醫院會在門診中主動辨識符合資格者並轉介篩檢。
- **問題**：並非所有符合資格的就診者都會被即時識別、主動轉介，導致高風險族群可能錯過早期發現機會。
- **價值**：若能以機器學習依病患的人口特徵、生活習慣與歷次篩檢紀錄，預先標出「可能篩檢異常」的高風險個案，可協助醫療人員在眾多候選名單中排定轉介優先順序，提升早期發現率、改善預後，同時不需額外的臨床檢驗資料即可運作（資料取得成本低、易於整合進既有看診流程）。
- **降低診間干擾、避免「轉介疲勞」**：若僅依國健署規定的資格條件，對所有符合資格者逐一列印癌篩轉介提示單，符合條件的就診者數量過於龐大，將對門診醫師造成極大的看診負擔；長期下來甚至可能讓醫師對高風險個案的衛教與轉介意願降低，形成「為了不漏掉而提醒所有人，最後反而沒有人被妥善轉介」的反效果。透過機器學習排序出真正最值得優先關注的個案，可大幅縮減需要醫師額外處理的名單量，讓「主動轉介」這件事真正落實，而不是淪為形式上的提醒、徒增診間負擔。

## 2. Project Objective（專案目標）

- **任務型態**：二元分類（Binary Classification）
- **預測目標**：`result`欄位 — 該次口腔癌篩檢結果是否「異常」（1=異常／0=正常）
- **輸入特徵**：病患的人口特徵（年齡、性別、是否原住民）、生活習慣風險因子（吸菸強度、嚼檳榔強度、自覺口腔不適）與歷次篩檢衍生紀錄（累計篩檢次數、上次篩檢結果、距上次篩檢年數），共 10 個特徵
- **最終目標**：比較至少 3 種課堂教過的機器學習演算法，找出在「不漏掉真正異常個案」（高 Recall）與「整體判別能力」之間表現最佳的模型，驗證僅憑基礎人口與生活習慣資料是否足以有效識別高風險個案，並評估整合進門診轉介流程的可行性

## 4. Data Preprocessing（資料前處理）

前處理流程定義於 `src/preprocess.py`，由 `run_preprocessing()` 依序執行：

1. **載入資料**（`load_data`）：以字串型別讀入 CSV，避免欄位型別被自動轉換出錯
2. **去識別化**（`remove_pii`）：以 `pd.factorize()` 將病歷號 `chMRNo` 轉為匿名流水號 `patient_id`，並刪除原始識別欄位
3. **日期換算與年齡計算**（`compute_age_and_year` + `parse_minguo_date`）：醫院系統日期為民國年格式（如 `1020305` = 民國102年3月5日），轉換為西元年後計算 `age = (篩檢日期 − 生日) // 365`，並取出 `screening_year`
4. **欄位重新命名**（`rename_columns`）：依 `RENAME_MAP` 將原始欄位名（`chSex`、`chSmoking` 等）轉為標準名稱
5. **類別編碼**（`encode_columns`）：
   - Label 編碼：`result` 代碼 `00`→0（正常），其餘→1（異常）
   - 原住民：`Y/N` → `1/0`
   - 數值型欄位（`smoking`、`betel_nut`、`oral_discomfort`、`gender`）轉為數值，無法轉換者設為 NaN
6. **缺漏值處理**（`handle_missing`）：
   - Label 缺漏 → 直接刪除該筆
   - `age`、`screening_year` → 以中位數填補
   - 生活習慣類欄位（`smoking`、`betel_nut`、`oral_discomfort`、`indigenous`）→ 缺漏視為「無此習慣」，填 0
   - `gender` → 以眾數填補
   - 基本特徵仍缺漏者 → 整列刪除
7. **衍生特徵工程**（`engineer_features`）：依病患 ID + 篩檢日期排序後，計算
   - `screening_count`：病患歷史累計篩檢次數（`groupby().cumcount()`）
   - `prev_result`：上一次篩檢結果（`shift(1)`，首次填 -1）
   - `years_since_last`：與上次篩檢的間隔年數（天數差 ÷ 365.25，首次填 0）
8. **資料切割**（`split_and_scale`）：以 `train_test_split(test_size=0.2, stratify=y, random_state=42)` 進行 80/20 分層抽樣切割，確保訓練集與測試集的異常比例一致（約 16,827 / 4,207 筆）
9. **標準化**（同上函式）：以 `StandardScaler` 做 Z-score 標準化；**僅在訓練集上 `fit`，再 `transform` 測試集**，避免資料洩漏（data leakage）
10. **類別不平衡處理（SMOTE 過採樣）**（`apply_smote`）：因異常樣本占比僅約 21%，對訓練集以 SMOTE 合成少數類別樣本至兩類數量平衡；**僅作用於訓練集，測試集維持原始分布**，確保評估結果反映真實情境

> 不平衡處理採「雙重保險」：訓練資料用 SMOTE 過採樣 + 四個模型皆設定 `class_weight="balanced"`，雙管齊下降低模型偏向多數類別（正常）的傾向。

## 5. Model Construction（模型建構）

選用四個分屬不同演算法類型的模型進行跨類型比較，超參數集中管理於 `config.py`，由 `src/models.py` 的 `build_models()` 建立：

| 模型 | 演算法類型 | 選用原因 | 主要超參數（`config.py`） |
|------|-----------|---------|--------------------------|
| **Random Forest** | 集成學習（Ensemble / Bagging） | 由多棵決策樹投票，抗雜訊、可處理特徵間非線性交互作用；可輸出 Feature Importance | `n_estimators=100`、`max_depth=10`、`class_weight="balanced"` |
| **Logistic Regression** | 線性模型 | 醫療研究最常用的基準模型，係數可直接解釋每個風險因子的貢獻方向與大小 | `C=1.0`、`max_iter=1000`、`class_weight="balanced"` |
| **SVM** | 核函數方法 | 以 RBF 核函數將資料映射至高維空間做非線性分類，與線性、樹狀模型形成對照 | `C=1.0`、`kernel="rbf"`、`probability=True`、`class_weight="balanced"` |
| **Decision Tree** | 樹狀模型 | 結構最直觀、最容易視覺化，適合在簡報中展示決策邏輯 | `max_depth=8`、`class_weight="balanced"` |

- 四個模型皆設定 `random_state=42` 以確保結果可重現
- `max_depth` 皆有限制（RF=10、DT=8），是刻意的設計選擇，目的是在模型複雜度與過擬合風險之間取得平衡（詳見第 7 節「模型比較」中的過擬合討論）
- 訓練（`train_all`）：四個模型各自在標準化＋SMOTE 後的訓練集上呼叫 `fit()`，測試集全程未參與訓練

## 6. Performance Evaluation（效能評估）

於測試集（4,207 筆，正常 3,305 / 異常 902）上計算 Accuracy、Precision、Recall、F1-score、AUC，並繪製 Confusion Matrix 與 ROC 曲線。

### 6.1 模型比較表（`model_comparison.csv` 實際輸出）

| 模型 | Accuracy | Precision | Recall | F1-score | AUC |
|------|----------|-----------|--------|----------|-----|
| Random Forest | 0.6855 | 0.3241 | 0.4302 | 0.3697 | 0.6300 |
| Logistic Regression | 0.6404 | 0.2841 | 0.4457 | 0.3470 | 0.6058 |
| SVM | 0.6477 | 0.2994 | 0.4800 | 0.3688 | 0.6249 |
| Decision Tree | 0.6910 | 0.3260 | 0.4135 | 0.3646 | 0.6160 |

指標意義（依本專題醫療情境的優先順序排列：**Recall > F1 > AUC > Precision > Accuracy**）：

- **Recall（敏感度）**：真正異常的個案中，被模型抓出的比例。本專題最重視此指標——漏掉真正異常者（False Negative）代表錯失早期發現機會，醫療代價最高
- **F1-score**：Precision 與 Recall 的調和平均，是程式中 `print_summary()` 用來選出最佳模型的依據
- **AUC**：模型整體區分「正常 vs 異常」能力的綜合指標，1.0 為完美、0.5 為隨機猜測
- **Precision（精確率）**：預測為異常者中真正異常的比例，反映「誤報」程度
- **Accuracy（準確率）**：在類別不平衡資料中參考價值最低（即使全部猜「正常」也能拿到約 78.6% 準確率）

### 6.2 Confusion Matrix 分析（`fig_confusion_matrix.png`）

四個模型在 4,207 筆測試集上的混淆矩陣（橫軸=預測、縱軸=實際；正常=3,305 筆，異常=902 筆）：

| 模型 | TN（正確判正常） | FP（正常誤判異常） | FN（異常誤判正常，漏報） | TP（正確抓出異常） |
|------|------|------|------|------|
| Random Forest | 2,496 | 809 | 514 | 388 |
| Logistic Regression | 2,292 | 1,013 | 500 | 402 |
| SVM | 2,292 | 1,013 | 469 | 433 |
| Decision Tree | 2,534 | 771 | 529 | 373 |

觀察重點：

- 四個模型的 **FN（漏報）都偏高**（373～529），代表約有四到六成的真正異常個案未被模型抓出，這是目前最大的弱點，也是後續調整決策門檻、特徵工程或取得更多臨床特徵時應優先改善的方向
- **SVM 的 FN 最低（469）、TP 最高（433）**，對應其 Recall 全模型最高（0.48）——在「醫療上不能漏掉異常」的優先順序下表現相對突出
- **Decision Tree 的 TN 最高（2534）、FP 最低（771）**，使其 Accuracy 全模型最高（0.691），但代價是 FN 也最高（529）、Recall 最低（0.4135）——典型的「準確率高但容易漏掉少數類別」現象，正是不能只看 Accuracy 的原因

### 6.3 ROC 曲線分析（`fig_roc_curve.png`）

- 四條曲線的 AUC 介於 0.606～0.630，皆明顯優於隨機猜測（對角虛線，AUC=0.5），但離「優良判別力」（一般認為 AUC ≥ 0.7）仍有差距
- 排序：Random Forest（0.630）> SVM（0.625）> Decision Tree（0.616）> Logistic Regression（0.606）
- 四條曲線彼此相當接近、區間多有重疊，顯示在目前僅有人口特徵與生活習慣資料的條件下，四種演算法對此問題的判別能力相近，模型選擇造成的差異小於資料本身資訊量的限制

## 7. Model Comparison（模型比較與分析）

### 7.1 哪個模型表現最好？為什麼？

這題的答案**取決於採用哪個指標**，這正是本專題分析的重點之一：

| 評估角度 | 最佳模型 | 數值 |
|---------|---------|------|
| F1-score（程式預設選模依據） | **Random Forest** | 0.3697 |
| Recall（醫療情境最優先） | **SVM** | 0.4800 |
| AUC（整體判別力） | **Random Forest** | 0.6300 |
| Accuracy（類別不平衡下參考價值最低） | Decision Tree | 0.6910 |

- 若依程式 `print_summary()` 採用的 **F1-score** 為準，**Random Forest** 是綜合表現最佳的模型（F1=0.3697，僅些微領先 SVM 的 0.3688），同時 AUC 也是四者中最高，顯示其在 Precision／Recall 的平衡與整體判別力上略勝一籌——這與其作為集成學習方法、能捕捉特徵間非線性交互作用（如吸菸與嚼檳榔並存時風險加乘）的特性相符
- 但若回到本專題明確設定的**醫療情境優先順序（Recall > F1 > ...）**，**SVM 才是 Recall 最高的模型**（0.48，較 RF 高出近 5 個百分點），代表它在「找出真正異常個案」這個最關鍵的任務上表現最突出。這提醒我們：**選模標準必須對齊應用場景的實際代價結構**，單一指標排名第一不代表它就是「最適合」的模型
- 整體而言，Random Forest 與 SVM 在 F1、AUC、Recall 上互有領先，皆可列為候選最佳模型；Logistic Regression 在各指標上均略遜於其他三者，Decision Tree 則是「Accuracy 最高、Recall 最低」的極端案例

### 7.2 是否有過擬合（Overfitting）現象？

- 本專題在建模階段已採取**預防性設計**：限制 Random Forest（`max_depth=10`）與 Decision Tree（`max_depth=8`）的樹深度，避免樹狀模型在訓練集上記憶雜訊而失去泛化能力
- 從目前測試集結果看，四個模型的 Accuracy（0.64～0.69）與 AUC（0.61～0.63）落在相近區間，沒有出現「某模型在測試集上效能異常突出或崩壞」的訊號，初步判斷過擬合程度有限
- 若要更嚴謹地驗證，建議後續**補上訓練集的同組指標**（目前 `evaluate.py` 僅計算測試集），比較訓練 vs. 測試的 Accuracy／F1 差距：差距越大，過擬合風險越高。這部分可作為書面報告中「分析深度」的加分項，亦可在簡報 Q&A 時主動說明已考慮但礙於時間未完整呈現

### 7.3 模型複雜度比較

| 模型 | 複雜度 | 訓練速度 | 可解釋性 | 備註 |
|------|--------|---------|---------|------|
| Logistic Regression | 低（線性模型，參數量 = 特徵數） | 快 | 高（係數可直接解讀風險方向與大小） | 假設線性決策邊界，難以捕捉特徵交互作用 |
| Decision Tree | 低～中（單一樹，深度限制為 8） | 快 | 高（樹狀結構可視覺化） | 單棵樹對資料雜訊敏感，容易產生不穩定的決策邊界 |
| SVM（RBF kernel） | 中～高（核函數將資料映射至高維空間） | 慢（本資料集 16,827 筆訓練樣本，RBF kernel 計算量大） | 低（高維空間中的決策邊界難以直觀解讀） | 對參數（C、gamma）敏感，需要調參 |
| Random Forest | 高（100 棵樹的集成） | 中（可平行化，`n_jobs=-1`） | 中（單樹不可解釋，但可輸出整體 Feature Importance） | 集成多棵樹降低單一樹的變異，但模型本身較難用單一規則說明 |

### 7.4 模型優缺點總表

| 模型 | 優點 | 缺點 |
|------|------|------|
| **Random Forest** | 整體判別力最佳（AUC、F1 領先）；可輸出特徵重要性，具醫療決策參考價值；對非線性關係與特徵交互作用的捕捉能力強 | 模型本身是黑箱（難以對單一病患說明「為什麼」判定為高風險）；訓練與儲存成本較高（`model_bundle.pkl` 須排除版控） |
| **Logistic Regression** | 結果最具可解釋性，係數可直接對應風險因子；訓練快、部署成本低，適合作為臨床基準模型 | 各項指標在四模型中均敬陪末座；無法捕捉特徵間的非線性交互作用 |
| **SVM** | Recall 最高，最符合「不能漏掉異常」的醫療優先序；以核函數處理非線性問題 | 訓練速度最慢；`probability=True` 進一步增加計算成本；參數調整（C、kernel、gamma）門檻較高、可解釋性低 |
| **Decision Tree** | 結構最直觀，最適合在簡報中展示決策邏輯給非技術背景的聽眾；訓練快 | Accuracy 雖最高，但 Recall 最低、最容易漏掉異常個案；單棵樹的決策邊界對資料變動較敏感（高變異） |

### 7.5 為什麼整體效能偏中等？

四個模型的 Accuracy 集中在 0.64～0.69、AUC 集中在 0.61～0.63，彼此差距不大。可能原因：

1. **特徵資訊量有限**：目前僅使用人口特徵、生活習慣強度與篩檢史等「結構化、易取得」的資料，未包含臨床檢驗、影像或病理等更直接的醫學指標，這類資料對「是否異常」的解釋力天生有上限
2. **類別不平衡**：即使以 SMOTE + `class_weight="balanced"` 雙重處理，異常樣本占比仍偏低（約 21%），模型要在「不過度誤報」與「盡量不漏報」間取捨並不容易
3. **標籤本身的雜訊**：篩檢「異常」涵蓋白斑、紅斑、紅白斑、疑似口腔癌等多種臨床表現，這些不同病灶的成因與風險因子組合可能不盡相同，被合併為單一二元標籤後，模型學習的「決策邊界」實際上是多種模式的混合，難度更高

> 以上分析正可呼應本專題提案的核心問題意識——**「僅憑人口特徵與生活習慣等基礎資料，是否足以有效識別口腔癌篩檢高風險個案？」**目前結果顯示：這些基礎資料確實提供了優於隨機猜測的判別力（AUC 0.6～0.63），但若要達到可獨立用於臨床決策的水準，仍需搭配更豐富的特徵或作為「初步篩選 / 排序輔助工具」而非「最終判定工具」使用。

## 8. Feature Importance Analysis（特徵重要性分析）

依據 Random Forest 的 `feature_importances_`（`fig_feature_importance.png`，數值為近似讀值）：

| 排名 | 特徵 | 重要性（約） | 說明 |
|------|------|------|------|
| 1 | **betel_nut（嚼檳榔強度）** | ≈ 0.32 | 全模型最重要的單一特徵，與口腔癌的醫學風險因子文獻高度吻合 |
| 2 | **age（年齡）** | ≈ 0.27 | 年齡是癌症類疾病常見的重要風險指標 |
| 3 | **smoking（吸菸強度）** | ≈ 0.17 | 與嚼檳榔合計貢獻接近一半的重要性，驗證「生活習慣是關鍵風險因子」的提案假設 |
| 4 | years_since_last（距上次篩檢年數） | ≈ 0.07 | |
| 5 | gender（性別） | ≈ 0.06 | |
| 6 | oral_discomfort（自覺口腔不適） | ≈ 0.04 | |
| 7 | prev_result（上次篩檢結果） | ≈ 0.03 | |
| 8 | indigenous（是否原住民） | ≈ 0.02 | |
| 9 | screening_count（累計篩檢次數） | ≈ 0.02 | 重要性最低 |

**分析重點**：

- **嚼檳榔強度、吸菸強度、年齡**三者合計貢獻約 76% 的重要性，與提案中「生活習慣相關因子（嚼檳榔、吸菸）將具有較高重要性」的假設**大致吻合**，但實際排名中 **age（年齡）的重要性高於原先預期**，甚至超過吸菸強度，僅次於嚼檳榔——顯示年齡作為癌症類疾病的基礎風險因子，其影響力不容低估
- 原先假設「歷史篩檢紀錄（上次篩檢結果 `prev_result`）將具有較高重要性」**未獲得驗證**：`prev_result` 與 `screening_count` 的重要性皆排在後段（約 0.02～0.03），顯示「這次篩檢是否異常」主要仍由病患的本質風險因子（生活習慣、年齡）決定，過去的篩檢紀錄相對是次要訊號
- 此排序結果具備**直接的臨床／政策意涵**：若醫療院所要建立「主動辨識高風險個案」的初步篩選規則，應優先掌握病患的嚼檳榔／吸菸習慣與年齡，這三項也恰好是目前篩檢資格認定（30 歲以上、有吸菸或嚼檳榔習慣）所依據的核心條件，與模型結果相互呼應
